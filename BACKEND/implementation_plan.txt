# Backend Login and Registration Implementation Plan

The goal is to create a Node.js backend in the `BACKEND` directory that handles the user authentication logic, verifies credentials, and stores new users. We will also integrate the frontend forms in `User.html` to send data to this new backend.

## User Review Required

> [!IMPORTANT]
> The requested task mentions storing new sign up data in `userdata.sql`. While we *will* add a feature to append new users to this backup file to satisfy your request, a live backend actually needs an active Database Server (like MySQL, XAMPP, or WAMP) to verify logins and read data efficiently. 
> Keep in mind that you will need to have a MySQL server running locally with the `watch_store` database created. If you do not have MySQL installed, we can switch to using `sqlite3` for an embedded local database. Let me know if you prefer SQLite!

## Proposed Changes

### Backend Setup

We will initialize a new Node.js project in the `BACKEND` folder and install necessary packages (Express, MySQL, bcrypt, CORS, and dotenv).

#### [NEW] [server.js](file:///c:/Users/Ishan/Desktop/E-commerce/E-Commerce-MINI-PROJECT/BACKEND/server.js)
The main Express application file that will contain:
- Configuration to connect to the active MySQL database `watch_store`.
- `POST /api/register`: Endpoint to accept user data, hash the password using `bcrypt`, insert the new user into the database, **AND** append an `INSERT INTO` statement into `userdata.sql`.
- `POST /api/login`: Endpoint to find a user by their email, compare the hashed password, and return a success message or error.

#### [NEW] [.env](file:///c:/Users/Ishan/Desktop/E-commerce/E-Commerce-MINI-PROJECT/BACKEND/.env)
A configuration file to securely store database credentials (e.g., `DB_HOST`, `DB_USER`, `DB_PASS`).

#### [NEW] [package.json](file:///c:/Users/Ishan/Desktop/E-commerce/E-Commerce-MINI-PROJECT/BACKEND/package.json)

### Frontend Integration

#### [MODIFY] [User.html](file:///c:/Users/Ishan/Desktop/E-commerce/E-Commerce-MINI-PROJECT/FRONTEND/User%20login-Register/User.html)
We will modify the existing `User.html` file to:
- Add `id` attributes to forms and input fields to easily capture data in JavaScript.
- Intercept the "form submit" events using `JavaScript` so we prevent the page from reloading.
- Use `fetch` API to send the login and sign-up data to our new `http://localhost:3000/api/register` and `http://localhost:3000/api/login` endpoints.
- Display success or error alerts based on the server response.
